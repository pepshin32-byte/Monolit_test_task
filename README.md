# Monolit — Система формирования заявок на материалы

**Структура проекта**
- `main.js` — основная точка входа (запускает приложение, взаимодействует с пользователем).
- `services/materialsService.js` — класс `MaterialsService` с логикой загрузки материалов, выбора региона/материала, генерации предложений и сохранения заявок.
- `data/` — папка с исходными данными; проект ожидает файл `materials.xlsx` (или `materials.json` в старых версиях).
- `orders/` — папка, куда сохраняются сформированные заявки в формате JSON.

Установка
1. Установите зависимости:

```bash
npm install
```
2. Запустите приложение

```bash
npm run main
```

Как это работает
- При запуске приложение загружает таблицу `data/materials.xlsx` (лист 1). Ожидаемые колонки: `name`, `category`, затем колонки цен для регионов (например, `СПБ`, `Москва`, `КРД`).
- Пользователь выбирает регион и материал через консольное меню (`readline-sync`).
- Если пользователь отказывается от оформления, скрипт предлагает более выгодную альтернативу или скидку и может сохранить заявку с предложением.
- Все сохранённые заявки пишутся в `orders/order_<timestamp>.json`.
